Functions

-A function is a piece of reusable code

-Functions address a key tenet of engineering: Don't Repeat Yourself, or DRY

-They are similar to objects or strings in that they can be passed into other functions as an argument

-If a variable is like a noun, a function would be a verb

Declaring a function

                  function sayHello () {
                    console.log("hello world");
                  }

  • We can also pass in arguments
    •                 function sayHello (name) {
                  console.log("hello " + name);
                }

    Calling a function

  • Since functions are reusable, they won't automatically run. You need to call them
    • sayHello();

  • And if you are using any arguments:
    • sayHello("Bob");

    Function Creation and Hoisting

  • Function Declaration (can be called before declaration, due to hoisting)
    •                 function sayHello (name) {
                  console.log("hello " + name);
                }

  • Function Expressions (must be defined before they are called)
    •                 let greeting = function (name) {
                  console.log("hello " + name);
                }

    Call a function as a Method

  • We haven't learned about Objects yet, but just know that a function can also be called from them
    •                 let person = {
                  name: 'Mr. Burns',
                  speak: function () {
                    console.log('Excellent...')
                  }
                }
              
              person.speak()
              => 'Excellent...'
  • You've used a method before. Log is a method of the Console
  • console.log("hello world")

    • The Return Statement

  • Sometimes we don't want to print to the console or update the DOM, but instead update a variable or even call another function
  • When we return something, it ENDS the function's execution and "spits out" what we are returning. We can then store this returned value in another variable... 
    •  
         function sum (x, y) {
           return x + y;
         }
         
         let z = sum(3, 4);
         => 7
  • Keep in mind that information we put after a return statement will not run
    •          function speak (words) {
           return words;
         
           // The following statements will not run:
         
           let x = “No one will”;
           let y = “see this message”;
         
           console.log(x + y)
         }

    Scope

    Scope is the set of variables you have access to

    Indentation may be a good clue as to where in scope you may be

    what will log from the code below?

             
      let a = "this is the global scope";
         
      function myFunction() {
        let b = "this variable is defined in the local scope";
      }
         
      myFunction();
         
      console.log(b);
  • But bear in mind, what is in the global can be accessible to local scopes farther down the scope chain
    •          let a = 1;
         
         function getScore () {
         
           var b = 2,
         
           c = 3;
         
              function add() {
         
                  return a + b + c;
         
              }
         
           return add();
         
         }
         
         getScore();
         
         => 6
