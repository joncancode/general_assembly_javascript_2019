The Power of JavaScript

  • JavaScript is used to add interactive elements like animations, buttons, and user input forms to the front-end. Increasingly, however, it is also being used to interact with APIs and databases and to perform back-end tasks
  • You can add Javascript directly to HTML and it will run in your browser
  • You can now run it outside of the browser as well, with Node.js, a JavaScript runtime environment
  • JavaScript can load content into the document if and when the user needs it, without the need to reload the entire page. This is commonly referred to as AJAX
  • JavaScript can determine your browser’s functional limitations and react accordingly. This is referred to as principles of unobtrusive JavaScript, or progressive enhancement
  • Unobtrusive JavaScript

    • JavaScript: the Bad Parts

    • JavaScript can be functionally flaky because the environment in which it operates is unknown: You don’t know the capabilities of computer on the receiving end of your Web page, you don’t know how busy the computer is with other functions (runs through a single thread)
    • JavaScript (or some of its functions) is occasionally turned off in browsers for security purposes or because it is often used for annoying pop-windows or intrusive advertising

    Node.js: Run JavaScript Anywhere!

    • Node was developed to run JS outside of the browser. We can run Node directly in our Terminal
    • Node uses an event-driven, non-blocking I/O model

    Git it Together

    What does Git do?

    • Primarily stores code, but can also store files, like Dropbox or Google Drive
    • Tracks changes so you can see who worked on what
    • Github makes it easy for multiple engineers to collaborate on the same project
    • Each GitHub project has a Git repository, or repo. Engineers joining a team start by "cloning" the repo. That is, they copy the Git repo from GitHub's cloud and save it in a local folder
    • The main, stable version of the codebase is on the default "branch" in Git, which is called master
    • Git cheatsheet

    Our First Repo

    • Go to your Github account and create a new repo called username.github.io where username is your username
    • In the Terminal, git init in the JS_goals folder
    • git init
    • git add .
    • git commit -m "first commit"
    • git remote add origin (your url)
    • git push -u origin master
    • Try visiting https://username.github.io/goals now
