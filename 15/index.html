Why React

The ~Virtual~ DOM

We've seen the DOM already. React introduces us to the Virtual DOM, a lightweight copy that only updates when it sees a difference in state

  • This is a huge concept in React - nothing is mutated, only updated
  • The 'state' is what the present data looks like. If you want to change the state, you need to update the virtual dom (think map() function)

    • Components

    What is a component

  • Components are basically JavaScript 'classes' that are attached to the HTML from a root div
  • Their re-usability is the main benefit
  • They work like Russian nesting dolls
  • Best practice: Create a folder for your components (aside from App and Index)
  • When a component has no need for State (we'll visit that later), we can even create a simpler 'Stateless' or 'Dumb' component as seen in this article
